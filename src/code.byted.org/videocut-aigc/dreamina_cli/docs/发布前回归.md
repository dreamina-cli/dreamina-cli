# 发布前回归

本文描述如何在发版前，用一条命令完成：

1. 构建目标产物
2. 检查新二进制版本信息
3. 跑固定样本的 `query_result` 旧新对比

对应脚本：

```text
scripts/preflight_release.sh
```

## 适用场景

这个脚本适合以下场景：

- 准备发布 `darwin/arm64`
- 准备发布 `darwin/amd64`
- 提交对齐修复后快速做一次回归
- 避免手工忘记更新 `dist/current/dreamina`

## 默认行为

默认执行：

1. 构建当前平台目标
2. 使用 `dist/current/dreamina` 作为新二进制
3. 输出 `dreamina version`
4. 调用 `compare_legacy_query_result.sh`
5. 使用 `testdata/legacy/query_result_submit_ids.txt` 作为固定样本清单

## 基本用法

当前平台一键回归：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh
```

同时构建并检查两个 Darwin 目标：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh \
  --target darwin/arm64 \
  --target darwin/amd64
```

只做比对，不重新构建：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh \
  --skip-build
```

只构建和看版本，不跑 `query_result` 对比：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh \
  --skip-compare
```

指定旧二进制和样本文件：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh \
  --old-bin /Users/saaspit/.local/bin/dreamina \
  --sample-file /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/testdata/legacy/query_result_submit_ids.txt
```

导出机器可读 JSON：

```bash
bash /opt/tiger/src/code.byted.org/videocut-aigc/dreamina_cli/scripts/preflight_release.sh \
  --skip-build \
  --json-output /tmp/dreamina_preflight.json
```

## 输出说明

脚本大致会输出三段：

### 1. 构建阶段

```text
==> preflight targets: darwin/arm64 darwin/amd64
==> build release targets
```

### 2. 版本检查

```text
==> new binary version
{
  "version": "ee56e05-dirty",
  "commit": "ee56e05",
  "build_time": "2026-04-05T..."
}
```

### 3. 回归对比

```text
==> compare legacy query_result samples
...
summary:
  total: 7
  exact_same: 3
  signed_url_only: 4
  different_resource: 0
  status_mismatch: 0
  queue_mismatch: 0
```

## 通过标准

当前建议的通过标准：

- `different_resource: 0`
- `status_mismatch: 0`
- `queue_mismatch: 0`

`scripts/preflight_release.sh` 现在会按上面三项自动判定退出码：

- 只要其中任意一项大于 `0`，脚本就会以非零状态退出
- `signed_url_only` 不会触发失败
- 这样可以直接接入 CI、发版脚本或本地自动化检查

`signed_url_only` 不作为失败条件，因为这通常表示：

- Dreamina 图片签名域名切换
- `vlabvod` 视频 URL 的分发 hash 或滚动值变化
- 服务端重复签发 URL

这类差异不应直接判定为客户端逻辑回归。
